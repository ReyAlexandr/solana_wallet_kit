# Security Policy

## Supported versions

Only the latest published version receives security fixes during the preview
phase.

## Reporting a vulnerability

Do not open a public issue for vulnerabilities that could expose recovery
phrases, private keys, backup contents, or secure-storage data.

Report security issues privately to `i.am.the.alexandr@gmail.com`. Include the
affected version, platform, reproduction steps, and impact. Do not include real
recovery phrases or private keys.

Please allow reasonable time to investigate and release a fix before public
disclosure.
